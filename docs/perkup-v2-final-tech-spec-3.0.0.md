# ☕ PerkUp v2.0 — Фінальне технічне завдання

**Версія:** 3.0.0 (фінальна)  
**Дата:** Березень 2026  
**Точки:** Mark Mall, ЖК Крона Парк 2, Парк Приозерний

## 1) Мета документа

Цей документ фіксує фінальні функціональні та технічні вимоги до PerkUp v2.0 як гейміфікованої екосистеми лояльності для мережі кав'ярень.

## 2) Базовий техстек

- Backend: Node.js + Fastify v4 + TypeScript
- DB: PostgreSQL + Prisma ORM
- Frontend: React 18 + Vite + Tailwind
- Realtime: Socket.io + Redis Adapter
- Черги: BullMQ + Redis
- Кеш/lock/rate-limit: Redis
- POS: Poster API v3 (окремі акаунти Крона/Приозерний)
- Оплата: WayForPay через Telegram Payments API
- Медіа: Telegram File Storage
- Принтери: ESC/POS (Mark Mall)

## 3) П'ять обов'язкових правил

1. Баланс балів завжди консистентний; краще втратити спін, ніж подвоїти нарахування.
2. Замовлення не губляться: при недоступності Poster — лише retry queue.
3. Меню між локаціями не змішується: `locationId` обов'язковий скрізь.
4. Геозона/бізнес-логіка валідуються тільки на бекенді.
5. `BOT_TOKEN` ніколи не потрапляє на клієнт; медіа лише через проксі API.

## 4) UX/продуктові вимоги (підсумок)

- Миттєвий splash screen (до React, без JS-залежностей).
- Онбординг новачка (3 екрани + мова + дефолтна локація + birthday opt-in).
- Sticky header з перемикачем точки (відстань + open/closed).
- Mark Mall режим самообслуговування (без кнопки онлайн-замовлення).
- Меню: skeleton loader, sticky категорії, фільтри/пошук/алергени.
- Кошик: realtime ціна модифікаторів, анімація додавання, редагування позицій.
- Оплата: часткова балами + карткою, атомарне списання балів.
- Чіткий flow помилок оплати без втрати кошика.
- Повний lifecycle замовлення, QR-видача, live ETA, бот-сповіщення.
- Чайові (до/після замовлення), no-show захист, повтор замовлення.

## 5) Операційна частина

- Зміни барист із PIN, чергою, автозакриттям.
- Busy mode (+ETA, обмеження черги).
- Офлайн-захист планшета у 3 рівні (heartbeat, SW, критичний оверлей).
- Автодрук стікерів у Mark Mall + кнопка повторного друку.

## 6) Лояльність і персоналізація

- Колесо фортуни: 1 спін/день, reset о 00:00 Kyiv, lock через Redis SET NX.
- Рівні Bronze/Silver/Gold/Platinum з щомісячним retention-контролем.
- Згоряння балів після 6 місяців неактивності з 2 попередженнями.
- Birthday flow (до ДН, в ДН, тиждень після).
- Комбо-набори (Bundle) як окрема сутність.
- Сегменти (VIP, sleeping, no-show тощо) та авто-тригери.

## 7) Безпека та надійність

- Idempotency key для бонусних операцій.
- Валідація webhook підписів (Poster + WayForPay).
- Retry/backoff для інтеграцій.
- Rate limit (Redis).
- Захист від читерства в іграх (серверна перевірка).
- Автоматизації через BullMQ (нагадування, SLA, expirations).

## 8) Мінімальний Definition of Done

- Запуск/навігація без білого екрану.
- Меню/кошик/оплата/повідомлення працюють за UX-вимогами.
- No-show, tips, bundles, repeat-order, loyalty-флоу працюють коректно.
- Техперевірки: `/health`, 401/429, webhook signatures, BOT_TOKEN hygiene.
- P95 завантаження ключових екранів < 2 секунд.

## 9) Дорожня карта

- Фаза 1–8 на 16 тижнів: від базової інфри та POS/payments до ігор, аналітики, QA та релізу.
